# Data Quality Report

## 목적

데이터 품질 리포트는 “데이터가 완벽하다”를 증명하는 문서가 아니라, 현재 데이터로 무엇을 계산할 수 있고 무엇은 보류해야 하는지 Codex와 사람이 같은 기준으로 판단하게 하는 문서입니다.

## 차단 기준

- 필수 컬럼 누락을 별칭으로 해결할 수 없음
- KODATA 조인 성공률이 설명 없이 급락
- `supportId` 생성 실패가 다수 발생
- `NOT_EVALUABLE`이 과도하게 높아 검토신호가 무의미
- 담당자/기업 모드 공개 범위를 리포트에서 확인할 수 없음
