# QUALITY_SCORE

도메인과 레이어별 품질 상태를 기록하는 파일입니다.

평가 후보:

- 테스트 신뢰도
- 문서 최신성
- 아키텍처 규칙 준수
- 관측 가능성
- 유지보수 위험
- 기술 부채

## 점수 기준

| 점수 | 의미 |
|---|---|
| A | 규칙이 자동화되어 있고 테스트와 문서가 최신입니다. |
| B | 주요 규칙은 지켜지지만 일부 자동화나 문서 보강이 필요합니다. |
| C | 동작은 하지만 구조, 테스트, 로그 중 하나 이상이 취약합니다. |
| D | 변경 위험이 높고 장애 분석이나 검증이 어렵습니다. |

## Java/Spring 평가 항목

- Controller가 얇은가?
- Service가 유스케이스 단위인가?
- Domain이 상태와 규칙을 소유하는가?
- Repository가 조회 의도를 드러내는가?
- Transaction 범위가 명확한가?
- 예외와 API 오류 응답이 표준화되어 있는가?
- 테스트가 도메인, 웹, JPA, 통합 계층별로 적절한가?
- 로그와 보안 기준이 지켜지는가?
