# Project Domain

## 한 줄 정의

DIVE 2026 부산테크노파크/KODATA 트랙의 목표는 신청기업의 객관 지표와 지원 이력을 한 화면에 모아 담당자와 평가위원이 평가 근거를 더 쉽게 이해하도록 돕는 선정 보조 웹 대시보드입니다.

## 최상위 원칙

- 우리는 평가를 자동화하지 않습니다.
- 우리는 평가의 근거를 더 쉽게 이해하도록 만듭니다.
- 모든 기능은 발제사가 내일부터 실제 업무에서 쓰고 싶은가로 판단합니다.
- 시스템은 선정, 탈락, 합격확률, 성공가능성을 판정하지 않습니다.
- 최종 판단은 담당자와 평가위원이 합니다.

## 발제사 문제

- 부산TP는 매년 약 200개 지원사업으로 약 1,200개 기업을 지원합니다.
- 부서별로 신청, 심사, 선정이 분절되어 중복 지원 파악이 어렵습니다.
- 서류와 발표만으로 기업의 실제 역량과 성장 가능성을 검증하기 어렵습니다.
- 지원사업의 실질적 성과를 확인하고 싶지만, 데이터가 흩어져 있습니다.

## MVP 기능

1. 기업 식별번호 또는 신청기업 목록 기반 조회
2. 기업 종합 스코어카드: 재무, 고용, 인증, 특허 요약
3. 부산TP 지원이력 타임라인
4. 검토 신호: 연계 가능성, 유사 목적 반복 가능성
5. 지원 이후 변화 확인: 지원 시점과 이후 관찰 지표를 함께 표시
6. 평가 근거 요약 카드: 출처, 수치, 검토 포인트를 한 장으로 정리
7. 신청 전 기업정보 점검: 기업용 PoC, 실제 도입 전 권한/인증 검토 필요

## 메시지 순서

1. 평가 근거를 강화한다.
2. 그 결과 더 적절한 지원이 연결될 수 있다.

“성장 연결”은 내부 컨셉명으로 쓸 수 있지만, 발표와 화면에서는 평가 근거를 먼저 말합니다.

## 금지

- 자동 선정
- 자동 탈락
- 합격확률 또는 선정가능성 수치
- 블랙박스 예측 모델
- 근거 사슬 없는 다음 지원 추천
- “지원 덕분에 성장했다” 같은 인과 확정 표현
- “지원 효과 검증”, “성과 측정” 같은 과장된 기능명
- 기업용 화면을 인증된 실제 서비스처럼 표현
- 평가위원 예상 질문을 기업에게 제공

## 권장 표현

| 피해야 할 표현 | 권장 표현 |
|---|---|
| 중복지원 기업 | 유사 목적 지원 이력 |
| 지원 효과 분석 | 지원 이후 변화 확인 |
| 성과 측정 | 성과 파악을 위한 1차 근거 |
| 왜 이 기업을 검토해야 하는지 | 이 기업을 검토할 때 확인해야 할 근거 |
| 합격을 위해 이렇게 준비하세요 | 신청 전 현재 데이터 상태를 확인해 보세요 |
| 평가위원 예상 질문 | 제출 전 확인사항 |
| 부실기업 | 재무 지속가능성 확인 필요 |

## 데이터 취급

- 기업 신용, 재무, 고용 데이터는 해커톤 목적에 한정된 민감 정보입니다.
- 외부 유출, 재배포, 상업적 이용을 금지합니다.
- 기업명, 대표자명, 법인등록번호, 사업자등록번호는 비식별/가명 처리될 수 있습니다.
- 현재 확정 가능한 결합키는 `기업일련번호`입니다.
- 사업자등록번호 입력은 실제 매핑 제공 여부 확인 전까지 구현/문구 모두 단정하지 않습니다.
