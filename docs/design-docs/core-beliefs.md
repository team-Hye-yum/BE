# Core Beliefs

- Codex가 읽을 수 없는 지식은 작업 중 존재하지 않는 지식으로 취급합니다.
- 긴 지침보다 작고 정확한 지도와 교차 링크를 우선합니다.
- 사람의 취향은 리뷰 코멘트로 끝내지 않고 문서 또는 자동화 규칙으로 남깁니다.
- 출력 속도가 빨라질수록 검증 루프와 정리 루프가 더 중요해집니다.
- Java/Spring 백엔드는 화려한 문법보다 예측 가능성, 변경 용이성, 테스트 가능성, 운영 안정성을 우선합니다.
- 도메인 규칙은 Service에 흩뿌리지 않고 도메인 객체와 정책 컴포넌트로 끌어올립니다.
- 운영에서 추적할 수 없는 코드는 아직 완성된 코드가 아닙니다.
