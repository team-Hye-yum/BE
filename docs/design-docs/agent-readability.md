# Agent Readability

에이전트 가독성은 Codex가 저장소 안에서 제품, 아키텍처, 운영 기준을 직접 추론할 수 있는 정도입니다.

기록할 후보:

- 도메인 용어
- 사용자 여정
- 아키텍처 예외
- 장애 대응 패턴
- 반복되는 리뷰 기준
- 검증 명령과 기대 결과

## Java/Spring에서의 에이전트 가독성

- 패키지 이름만 봐도 계층 책임이 드러나야 합니다.
- public Service 메서드는 유스케이스 문장처럼 읽혀야 합니다.
- 예외 타입과 오류 코드는 실패 의미를 드러내야 합니다.
- 테스트 이름은 조건과 기대 결과를 설명해야 합니다.
- 설정값은 하드코딩 대신 타입 안전한 설정 클래스로 모읍니다.
