# Execution Runbook

## 목적

샘플 데이터로 개발한 서비스를 현장 전체 데이터로 교체할 때 Codex와 개발자가 같은 순서로 움직이기 위한 런북입니다.

## 단계

1. 원천 파일 수령
2. 파일명, 시트명, 헤더 행 탐지
3. 필수 컬럼 존재 여부 검사
4. 컬럼 별칭 매핑 적용
5. 타입 변환과 단위 정규화
6. `기업일련번호` 기준 조인
7. 연도 반복 컬럼 long format 변환
8. 지원 건 `supportId` 생성
9. 분석 결과 사전 계산
10. 데이터 품질 리포트 생성
11. API/정적 JSON 샘플 비교
12. 발표용 데모 기업 재선정

## 중단 조건

아래 조건이면 기능 구현보다 데이터 확인을 먼저 합니다.

- 필수 시트가 없음
- `기업일련번호`가 없거나 대부분 비어 있음
- KODATA 조인 성공률이 예상보다 크게 낮음
- `1. 기업정보`의 지표/연도 헤더 구조가 샘플과 다름
- 0과 결측을 구분할 수 없음
- 담당자 모드와 기업 모드의 공개 범위를 결정할 수 없음
- 사업자등록번호 매핑이 없는데 입력 UI가 확정되어 있음

## 현장 교체 체크리스트

- [ ] 원천 파일 해시 또는 수령 시각 기록
- [ ] 시트별 행 수 기록
- [ ] 필수 컬럼 누락 여부 기록
- [ ] 컬럼 별칭 매핑 결과 기록
- [ ] 타입 변환 실패 건수 기록
- [ ] `기업일련번호` 중복/결측 기록
- [ ] KODATA 조인 성공률 기록
- [ ] 지원 건 중복 병합 수 기록
- [ ] `UNMAPPED` 지원품목 상위값 기록
- [ ] `SIGNAL_FOUND` / `NO_SIGNAL` / `NOT_EVALUABLE` 분포 기록
- [ ] 지원 이후 변화 확인 가능 비율 기록
- [ ] 정적 JSON 폴백 갱신 여부 확인

## 산출물

- 정제된 분석 입력 데이터

- 데모 기업 후보 목록
- 발표자료에 넣을 수치 변경 메모
