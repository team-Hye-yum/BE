# MERGE_PHILOSOPHY

처리량이 높은 에이전트 기반 개발에서는 병합 기준과 후속 수정 비용을 함께 고려합니다.

기록할 것:

- 반드시 차단해야 하는 게이트
- 후속 PR로 처리 가능한 문제
- 불안정 테스트 대응 방식
- 자동 병합 허용 조건

## Java/Spring 병합 기준

반드시 차단:

- 컴파일 실패
- 포매터/정적 분석 실패
- 구조 테스트 실패
- 인증/인가 누락
- 민감 정보 로그 노출
- Entity 직접 응답
- 표준 오류 응답 훼손
- DIVE 2026에서 자동선정/합격확률/인과확정 표현 추가
- 원천 엑셀 구조를 그대로 서비스 모델로 고정
- 담당자용 필드가 기업 모드로 노출

후속 PR 가능:

- 문서 예시 보강
- 일부 테스트 fixture 정리
- 중복 제거 리팩터링
- 품질 점수 B에서 A로 올리는 개선
