# FRONTEND

현재 하네스의 주 대상은 Java/Spring 백엔드입니다. 프론트엔드가 포함되는 경우에도 API 계약과 사용자 여정 검증은 백엔드 품질 게이트와 연결합니다.

## 백엔드와 맞물리는 기준

- API URL은 리소스 중심으로 설계합니다.
- 상태 코드는 일관되게 사용합니다.
- 검증 실패, 인증 실패, 권한 없음, 리소스 없음의 오류 포맷을 고정합니다.
- 페이징, 정렬, enum, 날짜, 금액 표현 방식을 문서화합니다.
- 프론트엔드가 Entity 내부 구조에 의존하지 않도록 Response DTO를 분리합니다.

## 계약 문서화

- 주요 API는 `docs/product-specs/`에 요청, 응답, 오류, 수용 기준을 기록합니다.
- 변경이 있으면 테스트와 문서를 함께 갱신합니다.
