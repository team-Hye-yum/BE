# Acceptance Criteria

기능 완료 조건은 다음 항목을 기준으로 작성합니다.

- 정상 요청이 기대 응답과 상태 코드를 반환합니다.
- 형식 검증 실패는 `400 Bad Request`로 표준 오류 응답을 반환합니다.
- 인증 실패와 권한 실패가 `401`과 `403`으로 구분됩니다.
- 존재하지 않는 리소스는 `404 Not Found`로 응답합니다.
- 도메인 상태 위반은 비즈니스 예외와 오류 코드로 표현됩니다.
- 주요 상태 변경은 추적 가능한 로그를 남깁니다.
- 민감 정보는 응답과 로그에 노출되지 않습니다.
- 도메인 단위 테스트, 필요한 slice/integration 테스트가 있습니다.

## DIVE 2026 공통 완료 조건

- 화면과 API 어디에도 자동 선정/탈락/합격확률이 없습니다.
- 지원 이후 변화에는 인과관계 미판단 문구가 포함됩니다.
- 모든 지표에는 출처가 표시됩니다.
- KODATA 미매칭과 결측은 0으로 표시하지 않습니다.
- 담당자 모드와 기업 모드의 응답 필드가 분리되어 있습니다.
- 신청 전 기업정보 점검에는 PoC와 실제 인증 필요성이 드러납니다.
- 데이터 품질 리포트에 기반해 데모 수치와 사례를 선택합니다.
- 전체 데이터 교체 후 `UNMAPPED`, `NOT_EVALUABLE`, 조인 실패 분포를 확인합니다.
